# 🚀 服务器部署指南

## 📋 快速开始（3 步部署）

### 第 1 步：配置服务器信息

```bash
# 复制配置模板
cp .deploy.config.example .deploy.config

# 编辑配置文件
vim .deploy.config
# 或 nano .deploy.config
```

**修改以下配置**：
```bash
SERVER_USER="root"              # SSH 用户名
SERVER_HOST="your-server-ip"    # 服务器 IP 或域名
SERVER_PORT="22"                # SSH 端口
SERVER_PATH="/var/www/naming-app"  # 部署路径
```

---

### 第 2 步：测试 SSH 连接

```bash
# 测试能否连接到服务器
ssh -p 22 root@your-server-ip

# 如果成功，说明 SSH 配置正确
# 按 Ctrl+D 或输入 exit 退出
```

---

### 第 3 步：执行部署

```bash
# 运行部署脚本
./deploy.sh

# 或使用生产环境
./deploy.sh production
```

## 🔧 服务器配置

### Nginx 配置（推荐）

**创建站点配置**：
```bash
sudo vim /etc/nginx/sites-available/naming-app
```

**配置内容**：
```nginx
server {
    listen 80;
    server_name your-domain.com;  # 或服务器 IP
    
    root /var/www/naming-app;
    index index.html mobile.html;
    
    # 日志
    access_log /var/log/nginx/naming-app-access.log;
    error_log /var/log/nginx/naming-app-error.log;
    
    # 静态文件缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # HTML 文件不缓存
    location ~* \.html$ {
        expires -1;
        add_header Cache-Control "no-cache";
    }
    
    # 支持 SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }
}
```

**启用站点**：
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/naming-app /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

---

### Apache 配置（备选）

**创建站点配置**：
```bash
sudo vim /etc/apache2/sites-available/naming-app.conf
```

**配置内容**：
```apache
<VirtualHost *:80>
    ServerName your-domain.com
    
    DocumentRoot /var/www/naming-app
    
    <Directory /var/www/naming-app>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
    
    ErrorLog ${APACHE_LOG_DIR}/naming-app-error.log
    CustomLog ${APACHE_LOG_DIR}/naming-app-access.log combined
</VirtualHost>
```

**启用站点**：
```bash
# 启用站点
sudo a2ensite naming-app

# 启用重写模块
sudo a2enmod rewrite

# 测试配置
sudo apache2ctl configtest

# 重载 Apache
sudo systemctl reload apache2
```

## 🔐 SSH 密钥配置

### 生成 SSH 密钥（如果没有）

```bash
# 生成新的 SSH 密钥
ssh-keygen -t ed25519 -C "naming-app-deploy"

# 按 Enter 使用默认路径
# 输入密码短语（可选，建议留空用于自动部署）
```

### 添加公钥到服务器

```bash
# 复制公钥到服务器
ssh-copy-id -p 22 root@your-server-ip

# 或手动添加
cat ~/.ssh/id_ed25519.pub | ssh root@your-server-ip "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

### 测试 SSH 连接

```bash
# 测试无密码登录
ssh -p 22 root@your-server-ip

# 如果成功，说明配置正确
```

## 📊 部署优化

### 1. 创建部署用户（可选）

```bash
# 创建专用部署用户
sudo useradd -m -s /bin/bash deploy

# 添加 SSH 密钥
sudo mkdir -p /home/deploy/.ssh
sudo cp ~/.ssh/id_ed25519.pub /home/deploy/.ssh/authorized_keys
sudo chown -R deploy:deploy /home/deploy/.ssh
sudo chmod 700 /home/deploy/.ssh
sudo chmod 600 /home/deploy/.ssh/authorized_keys

# 添加 sudo 权限（可选）
sudo usermod -aG sudo deploy
```

### 2. 配置防火墙

```bash
# UFW (Ubuntu)
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp   # HTTPS
sudo ufw enable

# Firewalld (CentOS)
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

### 3. 配置 SSL（推荐）

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx  # Ubuntu/Debian
sudo yum install certbot python3-certbot-nginx  # CentOS

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo crontab -e
# 添加：0 3 * * * certbot renew --quiet
```

## 🐛 故障排查

### 问题 1：SSH 连接失败

```bash
# 检查 SSH 服务状态
ssh root@your-server-ip "systemctl status sshd"

# 检查防火墙
ssh root@your-server-ip "sudo ufw status"

# 检查 SSH 端口
nmap -p 22 your-server-ip
```

### 问题 2：权限错误

```bash
# SSH 登录服务器
ssh root@your-server-ip

# 检查文件权限
ls -la /var/www/naming-app

# 修复权限
chown -R www-data:www-data /var/www/naming-app
chmod -R 755 /var/www/naming-app
```

### 问题 3：Nginx 502 错误

```bash
# 检查 Nginx 状态
sudo systemctl status nginx

# 查看错误日志
sudo tail -f /var/log/nginx/error.log

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 问题 4：部署后页面不更新

```bash
# 清除浏览器缓存
# Ctrl + Shift + R (Windows/Linux)
# Cmd + Shift + R (Mac)

# 或清除 Nginx 缓存
sudo nginx -s reload

# 检查文件时间戳
ssh root@your-server-ip "ls -la /var/www/naming-app/index.html"
```

## 📈 部署最佳实践

### 1. 版本管理

```bash
# 每次部署前创建标签
git tag -a v1.0.0 -m "生产环境部署"
git push origin v1.0.0

# 查看历史版本
git tag -l
```

### 2. 备份策略

```bash
# 部署前备份
ssh root@your-server-ip "cp -r /var/www/naming-app /var/www/naming-app.backup.$(date +%Y%m%d)"

# 保留最近 7 个备份
ssh root@your-server-ip "find /var/www -name 'naming-app.backup.*' -mtime +7 -delete"
```

### 3. 回滚方案

```bash
# 快速回滚到上一个版本
ssh root@your-server-ip "rm -rf /var/www/naming-app && cp -r /var/www/naming-app.backup.latest /var/www/naming-app"
```

### 4. 监控告警

```bash
# 安装监控工具
sudo apt install htop iotop iftop

# 查看实时日志
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log
```
